## ⚠️ ระบบนี้ทำได้ "พอใช้" ระดับไหน

### 🦠 วินิจฉัยโรคจากรูป (llava:7b)

**ความแม่น** ที่ผมประเมิน: **40-60%** สำหรับโรคที่อาการเด่นชัดมาก  
**สาเหตุ**: llava:7b ไม่เคย fine-tune กับโรคทุเรียนโดยเฉพาะ

| โรคที่น่าจะแม่น | โรคที่ AI งง |
|----|----|
| ใบจุดสีน้ำตาลชัดๆ | โรคยุคแรกๆ ที่อาการเบามาก |
| ผลเน่าทั้งผล | อาการร่วมหลายโรค |
| หนอนเจาะรู | ปัญหาขาดธาตุ vs โรค |

**สิ่งที่ทำได้**:
- ใส่ disclaimer ใน Flex card ทุกครั้ง: "ปรึกษานักวิชาการเกษตรหากไม่แน่ใจ" ✅
- ให้ผู้ใช้ส่งรูปหลายมุม → match KB จากหลาย signal
- ระยะยาว: fine-tune ด้วยรูปจริง (~200 รูป → ความแม่น 70-85%)

## 📡 แหล่งข้อมูลแต่ละตัว

### TMD API
- **ความแม่น**: 70% สำหรับฝน 24 ชม. / 60% สำหรับ 7 วัน
- **ปัญหา**: ต้องลงทะเบียน รอ 1-2 วัน อาจถูกปฏิเสธถ้าวัตถุประสงค์ไม่ชัด
- **Fallback**: Open-Meteo (ไม่ต้องสมัคร แต่แม่นน้อยกว่า ~60%)

### ราคาทุเรียน
- ผมเขียน scraper สำหรับ kasetprice + talaadthai เป็น **best-effort**
- เว็บเปลี่ยน HTML structure → scraper พังได้ (อาจต้องแก้ selector ทุก 1-2 เดือน)
- **Fallback**: ใช้ราคาประมาณตามฤดูกาล (ปลอดภัย แต่ไม่ใช่ realtime)
- **แนะนำ**: ใช้ MOC Open Data API (data.moc.go.th) เป็น source หลัก เสถียรกว่า — แต่ต้องเพิ่ม code parsing

### ราคายาง
- rakayang.net เปลี่ยนหน้าเว็บบ่อย — selector ของผมเป็นการเดา
- **ทางเลือก**: ใช้ thainr.com (กยท.) มี structure คงที่กว่า

### ราคาปุ๋ย  
- ผมใช้ **static reference data** (อ้างอิงเดือน เม.ย. 2569)
- ราคาจริงเปลี่ยนทุก 1-2 สัปดาห์ ตามตลาดน้ำมัน
- **แนะนำ**: ให้เกษตรกร update ราคาที่ตัวเองซื้อ → manual entry ใน Wiki ทุกเดือน

### YouTube
- ใช้ official API → เสถียร แต่จำกัด 10,000 quota/วัน (พอเหลือ)
- ต้องสมัคร Google Cloud + เปิด YouTube Data API v3

### TikTok
- **ไม่มี API ฟรี** — manual entry เท่านั้น
- ถ้าต้อง scrape จริง = เสี่ยงโดน TikTok block + ผิด ToS
- **แนะนำ**: ส่ง URL ใน LINE → ให้ Bot บันทึก (ฟีเจอร์ที่ยังไม่ได้ทำ)

## ⏱️ ปัญหา performance

### LINE 5-second timeout
LINE webhook ต้องตอบใน 5 วินาที — Ollama llava ใช้ 10-30 วินาที

**วิธีจัดการ**:
- ✅ ตอบ "กำลังประมวลผล..." ใน 5 วิ (code ผมทำแล้ว)
- ⚠️ ส่งผลจริงทีหลังผ่าน push API → กิน quota 200/เดือน
- หรือ ✅ เพิ่ม async queue (Cloudflare Queues — ฟรี 1M ops/เดือน)

ตอนนี้ code ผม fallback เป็นข้อความถ้า timeout → ผู้ใช้ต้องลองส่งใหม่

### Ollama รันบน CPU ช้า
- ไม่มี GPU → ตอบรูปใช้ 15-30 วินาที
- มี NVIDIA GPU → 2-5 วินาที

## 🔐 ความปลอดภัย

- ✅ LINE signature verification — ป้องกันคนปลอม webhook
- ✅ Cloudflare Tunnel — Ollama ไม่ต้องเปิด port public

## 💸 Hidden Costs

ที่บอกว่า "ฟรี" — มี implicit cost ดังนี้:

| รายการ | Cost ที่แท้จริง |
|--------|------------------|
| ค่าไฟ PC ที่บ้าน 24/7 | ~50-150 บาท/เดือน (ขึ้นกับสเปค) |
| Internet ที่บ้าน (ตัวเดิม) | 0 (ใช้ที่มีอยู่แล้ว) |
| Domain (option) | 300-500 บาท/ปี ถ้าอยาก URL สวย |
| Mobile data ของเกษตรกร | ~50-100 บาท/เดือน (ใช้ LINE) |

ถ้าไม่อยากเปิด PC 24/7:
- ปิด disease AI → เหลือแต่ ฝน/ราคา/ปุ๋ย → ใช้ Cloudflare Worker only = **ฟรี 100%**

## 🚧 สิ่งที่ระบบนี้ **ไม่ได้ทำ**

- ❌ ไม่ได้พยากรณ์ราคาด้วย ML จริงๆ — ใช้ rule-based ตามเดือนเท่านั้น
- ❌ ไม่ได้ตรวจดิน/น้ำในไร่ — ต้องมีอุปกรณ์ภาคสนาม
- ❌ ไม่ได้สั่งซื้อปุ๋ยให้อัตโนมัติ — แค่บอกราคา
- ❌ ไม่ได้แชร์ข้อมูลระหว่างเกษตรกรในชุมชน (1 Bot = 1 สวน)
- ❌ ไม่ได้ตอบคำถามที่อยู่นอก scope เช่น "ขายที่ไหนดี" "ส่งออกได้ไหม"

ถ้าต้องการฟีเจอร์เหล่านี้ → ต่อยอดได้
